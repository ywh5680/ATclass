### 实践作业

#### 作业 1：Git 基础操作

**目标**：掌握 Git 的基本操作

**要求**：
1. 创建一个新的 Git 仓库
2. 添加一些文件并进行提交
3. 创建和切换分支
4. 合并分支
5. 查看提交历史

**步骤**：
```bash
# 1. 创建仓库
mkdir my-git-project
cd my-git-project
git init

# 2. 添加文件
echo "# My Git Project" > README.md
echo "console.log('Hello Git!');" > app.js

# 3. 提交文件
git add .
git commit -m "Initial commit"

# 4. 创建分支
git checkout -b feature/new-feature

# 5. 在分支上修改
echo "// New feature" >> app.js
git add app.js
git commit -m "Add new feature"

# 6. 合并分支
git checkout main
git merge feature/new-feature

# 7. 查看历史
git log --oneline
```

#### 作业 2：远程仓库操作

**目标**：掌握远程仓库的使用

**要求**：
1. 在 GitHub 上创建远程仓库
2. 配置 SSH 密钥
3. 推送代码到远程仓库
4. 从远程仓库拉取更新
5. 解决合并冲突

**步骤**：
```bash
# 1. 配置 SSH 密钥
ssh-keygen -t ed25519 -C "your.email@example.com"
# 将公钥添加到 GitHub

# 2. 添加远程仓库
git remote add origin git@github.com:username/repository.git

# 3. 推送代码
git push -u origin main

# 4. 创建功能分支
git checkout -b feature/remote-test
echo "// Remote feature" >> app.js
git add app.js
git commit -m "Add remote feature"

# 5. 推送分支
git push -u origin feature/remote-test

# 6. 创建 Pull Request
# 在 GitHub 上创建 PR 并合并
```

#### 作业 3：分支管理策略

**目标**：理解不同的分支管理策略

**要求**：
1. 实现 Git Flow 工作流
2. 实现 GitHub Flow 工作流
3. 比较两种工作流的优缺点
4. 选择适合项目的工作流

**步骤**：
```bash
# Git Flow 工作流
# 1. 创建开发分支
git checkout -b develop
git push -u origin develop

# 2. 创建功能分支
git checkout -b feature/user-auth
# 开发功能...
git push -u origin feature/user-auth

# 3. 合并到开发分支
git checkout develop
git merge feature/user-auth
git push origin develop

# 4. 创建发布分支
git checkout -b release/v1.0.0
# 准备发布...
git checkout main
git merge release/v1.0.0
git tag v1.0.0
git push origin main
git push origin v1.0.0

# GitHub Flow 工作流
# 1. 从主分支创建功能分支
git checkout -b feature/simple-flow
# 开发功能...
git push -u origin feature/simple-flow

# 2. 创建 Pull Request
# 在 GitHub 上创建 PR

# 3. 代码审查后合并
# 通过 PR 合并到主分支
```

#### 作业 4：版本发布管理

**目标**：掌握版本发布流程

**要求**：
1. 创建版本标签
2. 生成发布说明
3. 自动化发布流程
4. 实现回滚机制

**步骤**：
```bash
# 1. 创建发布分支
git checkout -b release/v1.0.0

# 2. 更新版本号
npm version patch --no-git-tag-version

# 3. 更新 CHANGELOG
cat >> CHANGELOG.md << 'EOF'

## [1.0.1] - 2024-01-15

### 新增
- 用户认证功能
- 数据导出功能

### 修复
- 修复登录验证问题
- 修复数据导出错误
EOF

# 4. 提交更改
git add .
git commit -m "Prepare release v1.0.1"

# 5. 合并到主分支
git checkout main
git merge release/v1.0.1

# 6. 创建标签
git tag v1.0.1

# 7. 推送
git push origin main
git push origin v1.0.1

# 8. 删除发布分支
git branch -d release/v1.0.1
```

### 综合项目

#### 项目：个人博客系统

**项目描述**：开发一个基于 Git 版本控制的个人博客系统

**功能要求**：
1. 文章管理（增删改查）
2. 用户认证
3. 评论系统
4. 标签分类
5. 搜索功能

**技术栈**：
- 前端：HTML, CSS, JavaScript
- 后端：Node.js, Express
- 数据库：SQLite
- 版本控制：Git

**开发流程**：
```bash
# 1. 项目初始化
mkdir personal-blog
cd personal-blog
git init

# 2. 创建项目结构
mkdir src public docs tests
touch README.md package.json .gitignore

# 3. 初始化项目
npm init -y
npm install express sqlite3

# 4. 创建基础文件
echo "# Personal Blog" > README.md
echo "node_modules/" > .gitignore
echo "*.log" >> .gitignore

# 5. 提交初始版本
git add .
git commit -m "Initial project setup"

# 6. 创建功能分支
git checkout -b feature/user-authentication
# 开发用户认证功能...

git checkout -b feature/article-management
# 开发文章管理功能...

git checkout -b feature/comment-system
# 开发评论系统...

# 7. 合并功能
git checkout main
git merge feature/user-authentication
git merge feature/article-management
git merge feature/comment-system

# 8. 发布版本
git tag v1.0.0
git push origin main
git push origin v1.0.0
```
